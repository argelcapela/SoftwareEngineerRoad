--COMMIT: alterar;
--comitar é modificar;
--GITHUB: Repositorio remoto;
--GIT: Repositório local;
--branch master: é a pasta principal;
--untracked files: são arquivos o GIT não tá vendo;

-- GIT BASH: terminal específico para GIT com mais comandos;




COMMANDS
git config --list
git config --global user.name "MEU NOME"
git config --global user.email "ARGEL@HOTMAIL.COM"

git init <URL>						// inicia o GIT (versionamento) do seu projeto;
git status 						// mostra os arquivos que o GIT tá olhando;
	
git add <NOME>						// faz o GIT monitorar esse arquivo; só se usa quando o projeto tem arquivos NOVOS, dps só o [commit -am]; Sempre que um commit é feito, é como se o GIT parasse de monitorar. A cada modificação ou alteração no projeto tem que ser usado o add dnv, então por isso é mais fácil usar o commit -am que já adianta os dois em um só. MAS quando novos arquivos são adicionados no projeto tem que digitar o [git add -A] no unha senão não funciona;
git add -A   						// add todos os arquivos do diretorio;

git commit -m "Primeiro Commit"	  			// envia pro meu GIT, as alterações do projeto;
git log							// mostra todos os commits feitos no branch

git commit -am "Segundo Commit!"			// envia pro meu GIT todas as alterações do projeto, sem precisar usar git add -A de novo, toda vez que houver uma modificação;






git reset --mixed   <hash completo do commit ou só 7 primeiros números>			// 
git reset --soft    <hash completo do commit ou só 7 primeiros números>			// 
git reset --hard    <hash completo do commit ou só 7 primeiros números>			// todos os arquivos voltam RADICALMENTE, para o estado em que se encontravam naquele COMMIT. Não é recomendado usar quando trabalhando com equipes, CUIDADO! 



git help <command>					// obtém ajuda com o comando;




git branch						// lista todos os branches, o que tiver * é o atual;
git branch <NOME DO BRANCH NOVO>			// cria um branch novo a partir do estado atual do projeto; a ideia é desenvolver uma nova funcionalidade sem alterar o MASTER, sem o projeto "ONGOING" sair do AR; depois de terminar se der certo eu só mando as alterações para o BRANCH MASTER;
git checkout <NOME DO BRANCH>				// troca de branch; os arquivos e alterações são feitos automaticamente; ISSO É TOP!



git diff 						// mostra o que foi modificado nos arquivos do repositório;
git diff --name-only				 	// nome dos arquivos que foram alterados;
git diff <NOME DO ARQUIVO>				// mostra tudo o que foi modificado nesse arquivo;
git checkout HEAD -- <NOME DO ARQUIVO>			// restaurar o arquivo para sua última versão comitada;




$ls -al ~/.ssh								// verifica se existem chaves ssh; se o diretorio não existe é pq não tem nenhuma;
$ssh-keygen -t ed25519      -C "argelcapeladossantos@gmail.com"		// gera uma nova chave ssh guardada em ..//user//ssh//	
$ssh-keygen -t rsa -b 4096  -C "argelcapeladossantos@gmail.com"	        // gera uma nova chave ssh guardada em ..//user//ssh//	
*depois de criar as chaves, abri a pública, copia o código, e lá no site do github cria uma chave ssh, cola o codigo lá e escolhe um titulo;*





git remote add origin https:\\www.github.com\capelaplus.com		// faz a conexão entre o repositorio local e o remoto GITHUB;
git remote -v								// visualiza as conexões do repositório atual;






#antes de dar um push, sempre dar um pull para evitar problemas!#
git push -u origin master						// passa todos os arquivos para o repositório remoto;







*criando um arquivo chamado .gitignore sem extensão, todos os arquivos que estiverem listados lá, não apareceram no repositorio remoto! Mt útil em casos de informações confidenciais que não podem ser exibidas para qualquer um no projeto;*





git revert --no-edit <ID DO COMMIT>					// salvo o estado atual em um commit chamado revert, e reseta o repositório para o último comite feito anteriormente;





git push origin :<nome branch>						// deleta o branch remoto;
git branch -D <nome branch>                                             // deleta o branch local;





git pull <origem> <branch que está indo>				// pega os arquivos novos do repositorio remoto para o seu local;




git clone <URL PROJETO NO GITHUB>					// copia todo o repositório de um projeto para a sua maquina;
